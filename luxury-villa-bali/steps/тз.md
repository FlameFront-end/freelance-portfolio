# Luxury Villa Bali (Concept UI)
## Premium Real Estate / Villa Booking Landing — Full Technical Specification

---

## 0. Цель проекта

Сделать визуально премиальный сайт для аренды luxury-виллы на Бали, который выглядит как коммерческий продукт агентства.

Проект должен:
- продавать визуалом (крупные фото, типографика, “воздух”)
- включать параллакс и мягкие анимации
- иметь галерею с лайтбоксом
- иметь форму бронирования (UI-only, без бэка)
- быть NDA-safe (никаких реальных компаний/логотипов)

Формат: landing + несколько секций, можно как single-page.
Название (пример): **AURA VILLA — Bali**

---

## 1. Стек

Обязательно:
- React + TypeScript (Vite)
- SCSS Modules 
- Framer Motion (анимации)
- Библиотека галереи (одна):
  - yet-another-react-lightbox (предпочтительно)
  - или react-photo-gallery + lightbox
- (Опционально) react-intersection-observer

Запрещено:
- реальные бренды/агентства
- реальные логотипы booking/airbnb и т.п.

## 2. Deliverables

1) Репозиторий + README  
2) Деплой (Vercel/Netlify)

## 3. Визуальная стилистика

### 3.1 Визуальное ощущение
Ассоциации:
- luxury hospitality
- calm, warm, premium
- editorial design
- minimalist elegance

Принцип:
- меньше элементов
- больше воздуха
- крупные фото
- типографика решает

### 3.2 Цветовая система

Base:
- Background: `#0B0B0C` (dark) или `#FAF7F2` (light, предпочтительно для luxury)
Выбрать один основной стиль:

Вариант A (Luxury Light, рекомендовано):
- `--bg: #FAF7F2`
- `--panel: #FFFFFF`
- `--text: #141414`
- `--text-2: #4B4B4B`
- `--muted: #8A8A8A`
- `--border: rgba(20,20,20,0.10)`
- `--accent: #B08D57` (warm gold)
- `--accent-soft: rgba(176,141,87,0.14)`

Вариант B (Luxury Dark):
- `--bg: #0B0B0C`
- `--panel: #121214`
- `--text: #F5F2EC`
- `--text-2: rgba(245,242,236,0.75)`
- `--muted: rgba(245,242,236,0.55)`
- `--border: rgba(245,242,236,0.10)`
- `--accent: #C7A46A`
- `--accent-soft: rgba(199,164,106,0.16)`

Правила:
- акцент только на CTA и small highlights
- никаких ярких цветов

### 3.3 Шрифты (очень важно для luxury)
Нужно 2 шрифта:

1) Display / Editorial (заголовки):
- Playfair Display / Cormorant Garamond / DM Serif Display

2) UI / Body (текст):
- Inter / Manrope / DM Sans

Размеры:
- H1: 44–64px (desktop), 32–40px (mobile)
- H2: 28–36px
- Body: 16px
- Meta: 12–13px
Line-height:
- заголовки 1.05–1.15
- текст 1.6

## 5. Структура страницы `/` (Landing)

### 5.1 Header (sticky, прозрачный поверх hero)
- Лого: “AURA VILLA”
- Навигация: Villa, Gallery, Amenities, Location, Reviews, Book
- CTA: “Check availability”
- На scroll фон становится solid + blur (если dark) или solid white (если light)

### 5.2 Hero (экран 1)
Требования:
- full height (min 90vh)
- фоновое фото (крупное)
- overlay gradient (чтобы текст читался)
- H1 + subtitle + 2 кнопки:
  - Primary: “Check availability”
  - Secondary: “Explore villa”
- Блок “Quick facts” (3–5 меток): 3BR / 6 guests / private pool / Pererenan

Анимация:
- плавное появление текста (fade + small slide)
- без лишних эффектов

### 5.3 Parallax Section (Story)
Секция с параллаксом:
- фоновое фото виллы (параллакс при скролле)
- поверх короткий текст:
  - “Designed for privacy”
  - “Sunset terrace”
  - “Natural materials”

Реализация параллакса:
- вариант A: CSS background-attachment: fixed (если подходит)
- вариант B: JS через scroll position (не тяжело), transform: translateY

### 5.4 Highlights (3 карточки)
3 карточки с фото (или иконкой):
- Private pool
- Tropical garden
- Sunset terrace

Стиль:
- крупные изображения
- минимальный текст
- hover: лёгкое затемнение/подсветка, без scale

### 5.5 Gallery (обязательная)
- сетка 2–3 колонки (desktop), 1 колонка (mobile)
- минимум 10 изображений (можно placeholders)
- при клике открывается lightbox
- в lightbox:
  - next/prev
  - counter
  - keyboard navigation

Состояния:
- skeleton loading (эмуляция 600–900ms)

### 5.6 Amenities (список + инфографика)
Секция:
- список удобств (12–16 пунктов)
- разбить на группы:
  - Comfort
  - Kitchen

  - Services
- можно добавить 2 “feature cards” с иконками:
  - Fast Wi-Fi
  - Daily housekeeping

### 5.7 Location (карта как UI)
Без реальной карты (чтобы не тянуть API):
- сделать “map-like” блок (стилизованный)
- рядом список:
  - Beach: 8 min
  - Cafes: 3 min

Опционально:
- кнопка “Open in Maps” (просто ссылка-заглушка)

### 5.8 Reviews
- 3 отзыва (карточки)
- имя + страна + оценка (stars)
- текст 2–3 строки

### 5.9 Final CTA
Большой баннер:
- “Ready for your Bali stay?”
- кнопка “Book now”

### 5.10 Footer
- ссылки
- “© 2026 AURA VILLA”

## 6. Форма бронирования (UI-only) — обязательная логика на фронте

Форма должна быть доступна:
- либо как секция `/book`
- либо как модал по кнопке “Check availability”

### Поля:
- Check-in date (datepicker или input type=date)
- Check-out date
- Guests (stepper 1–6)
- Name
- Email
- Notes (optional)

### Валидация:
- check-out > check-in
- guests 1..6
- email валиден
- required: check-in, check-out, guests, name, email

### Поведение:
- submit показывает “Booking request sent” (toast/inline state)
- сохранять последнее значение дат/гостей в localStorage (опционально)
- никакого бэка, только UI state

### Доп. блок цены (очень усиливает)
Справа/снизу “Booking summary card”:
- Price per night (mock)
- Nights count (рассчитать)
- Cleaning fee (mock)
- Total (рассчитать)
- CTA “Request booking”

## 7. Анимации и UX

Обязательные:
- Smooth scroll к секциям
- Fade-in секций при появлении (intersection observer + motion)
- Header меняет стиль после скролла
- Lightbox плавно открывается
- Модал брони (если есть) — плавно открывается

Запрещено:
- bounce
- чрезмерные движения
- тяжёлые 3D эффекты

---

## 8. Адаптивность

Обязательно:
- Mobile 375px
- Tablet 768px
- Desktop 1280px+

Правила:
- hero всегда выглядит “дорого”
- gallery на mobile — 1 колонка
- booking summary card:
  - desktop: справа sticky
  - mobile: внизу страницы после формы

## 9. Performance

- изображения через оптимизацию (next/image если Next.js)
- lazy-load галереи и lightbox
- минимизировать тяжёлые зависимости
- Lighthouse Performance целиться >= 90

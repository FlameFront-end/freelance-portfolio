# ТЗ: Премиальный Landing для digital-продукта (Concept Project)

## 0) Цель
Сделать визуально «дорогой» одностраничный лендинг (SaaS / digital product), который выглядит как коммерческий проект, но не использует материалы реальных клиентов. Результат — готовый деплой + репозиторий + набор скриншотов для портфолио.

## 1) Технологии
- Next.js (App Router)
- TypeScript
- TailwindCSS
- Framer Motion
- (Опционально) Three.js или Spline embed для 3D в hero

## 2) Результаты (deliverables)
1) Репозиторий с исходниками и README
2) Деплой (Vercel)
3) Страница `/` (landing) + доп. страницы:
   - `/pricing`
   - `/docs` (простая страница-заглушка)
4) Набор скриншотов (desktop + mobile), минимум 6 штук
5) Lighthouse Performance >= 90 на главной (без фанатизма, но без тяжелых блокирующих скриптов)

## 3) Основная концепция дизайна
- Тёмная тема (основная)
- Стиль: premium tech / neo glass / soft blur
- Элементы:
  - градиенты (фоновые, мягкие)
  - blur (glassmorphism)
  - тонкие border (1px, полупрозрачные)
  - мягкие тени
- Типографика: 1 display font (например Inter / Manrope / Satoshi-like) + четкая иерархия заголовков

## 4) Контент (заглушки, можно использовать)
Название продукта: **Nebula**
Слоган: **Ship premium UI faster.**
Описание: **Nebula is a component platform for teams that build modern web apps. Animations, themes, and layouts included.**

CTA кнопки:
- Primary: `Get started`
- Secondary: `View demo`

## 5) Структура страницы `/` (обязательные секции)
### 5.1 Header (sticky)
- Лого слева (Nebula)
- Навигация: Features, Showcase, Pricing, Docs
- Справа кнопки: `Login` (secondary), `Get started` (primary)
- Sticky + backdrop blur, легкая полупрозрачность

### 5.2 Hero (экран 1)
- Большой заголовок (2 строки)
- Подзаголовок 1–2 строки
- 2 CTA кнопки
- Визуальный акцент справа/по центру:
  - вариант A: 3D объект (Three.js)
  - вариант B: Spline embed (предпочтительно быстрее)
- Фоновый градиент + “glow” вокруг 3D
- Легкая анимация появления текста/кнопок (Framer Motion)

### 5.3 Social Proof
- Строка с логотипами (можно текстовые «Acme, Vertex, Nova…» или простые SVG-заглушки)
- Текст: `Trusted by teams shipping weekly`

### 5.4 Features (3–6 карточек)
- Сетка карточек (glass)
- Каждая карточка:
  - иконка (SVG / lucide-react)
  - заголовок
  - описание 1–2 строки
- Анимация появления карточек при скролле (stagger)

Пример фич:
- Real-time components
- Theme system
- Motion presets
- Accessible by default
- Layout blocks
- Developer-friendly API

### 5.5 Showcase (мини-галерея)
- 2–3 “скриншота” интерфейса (можно сделать фейковые UI блоки)
- Можно сделать вкладки “Dashboard / Billing / Analytics”
- Переключение вкладок меняет контент (анимация)

### 5.6 Testimonials (2–3)
- Карточки с отзывами
- Имя + роль + “компания”
- Фото не нужно (можно initials-аватар)

### 5.7 Pricing teaser (ссылка на /pricing)
- 3 тарифных карточки:
  - Starter, Pro, Team
- Кнопка `See full pricing` ведет на /pricing

### 5.8 FAQ (аккордеон)
- 5 вопросов
- Аккордеон с плавной анимацией

### 5.9 Final CTA
- Большой блок-баннер (glass + glow)
- Текст + кнопка `Get started`

### 5.10 Footer
- Линки: Product, Company, Legal, Social
- Copy: `© 2026 Nebula`

## 6) Страница `/pricing`
- Заголовок + краткое описание
- Переключатель Billing: Monthly / Yearly (анимация, меняет цены)
- 3 карточки тарифов:
  - Starter: $19 / $190
  - Pro: $49 / $490 (highlight)
  - Team: $99 / $990
- Таблица сравнения (минимальная)
- CTA на каждом тарифе

## 7) Страница `/docs` (простая, но красивая)
- Sidebar слева (3–6 пунктов)
- Контент справа:
  - Getting Started
  - Installation
  - Usage
  - Theming
- Небольшие code-block элементы (статичные, без реального API)

## 8) Анимации (обязательные)
- Hero: fade/slide-in текста, кнопок и 3D блока
- Features: stagger карточек при входе в viewport
- Tabs в Showcase: плавная смена контента
- FAQ: раскрытие/сворачивание с height animation
- Navbar: sticky blur + легкая смена opacity при скролле

Требование: анимации не должны лагать (duration 0.35–0.6s, ease-out).

## 9) 3D (варианты реализации)
### Вариант A (быстро): Spline
- Вставить Spline embed в hero
- Ленивая загрузка (динамический импорт или условная отрисовка)
- Фолбэк: статичный градиентный блоб / SVG

### Вариант B (сложнее): Three.js
- Легкий объект (torusKnot / sphere) + простейший свет
- Минимальный вес, без тяжелых текстур
- Отключить на mobile или упростить

## 10) Адаптивность
- Обязателен responsive:
  - Mobile (375px)
  - Tablet (768px)
  - Desktop (>= 1280px)
- На mobile:
  - 3D можно скрыть или заменить на легкий блоб
  - меню превращается в бургер

## 11) Производительность
- Использовать next/image для изображений
- Минимизировать heavy scripts
- 3D: lazy load
- Шрифты через next/font

## 12) Структура проекта (рекомендация)
- `app/` (App Router)
- `components/` (UI блоки: Navbar, Hero, FeatureGrid, PricingCards, FAQ, Footer)
- `lib/` (helpers, constants)
- `content/` (тексты, массивы features/pricing/faq)
- `styles/` (globals, tailwind)

## 13) Данные (без бэка)
- Все данные — статические массивы в коде.
- Никаких реальных брендов/компаний/логотипов клиентов.

## 14) Acceptance Criteria (критерии готовности)
- Визуально выглядит как коммерческий лендинг уровня premium
- Все секции присутствуют и читаемы
- Анимации работают корректно и плавно
- `/pricing` и `/docs` существуют и выглядят в той же стилистике
- Деплой работает, без критических ошибок в консоли
- Скриншоты для портфолио подготовлены
